ValueLens

Installed

Value & ROI

—

One Power BI template for every Copilot and agent adoption signal. ValueLens turns Purview audit logs, licensing data, and org structure into a defensible ROI narrative — hours saved, assisted value, adoption, maturity, and governance across your whole Copilot estate, aligned to Microsoft's Frontier Firm framework.

What it shows

Every Copilot interaction is classified into an AI Task, mapped to a research-sourced time baseline, summed into Hours Saved, and multiplied by hourly rates into Assisted Value — a value model grounded in published research from Microsoft Research, MIT, NBER, BCG/Harvard, McKinsey, Forrester, IDC and others. On top of that model, ValueLens measures: quantified value in hours and dollars; your position on the Frontier Firm journey from human + Copilot, to human + agent, to agent-run workflows; value broken down by function — Sales, HR, IT, Legal, Finance, Marketing, Customer Service — with task-level attribution; user maturity from Beginner through Proficient; and a full business case with projected annual value, ROI multiple, and license investment net.

What's inside

Fifteen report pages covering the full story: User Activation, Adoption & Reach, Activity & Value, Usage Maturity, Leaderboards, Agent Governance, User Feedback, License Readiness, and Heatmap Trend — plus three Copilot Studio pages in the extended build (Credits Consumed, Agent Evaluation, Topic Analysis) and three appendices (Key Concepts, Glossary, and a Signal Table that traces raw signals through to value as an audit trail).

Data requirements

Three core sources: Copilot interaction audit logs from Microsoft Purview, licensed users from the Microsoft 365 Admin Center, and org data from Microsoft Entra. Optional sources — Agent 365 exports, Cowork/Work IQ consumption, credit billing, product feedback, and Copilot Studio transcripts — light up additional pages and are gated by toggles; the dashboard works fully without them.

Get this lens

ValueLens ships in three deployment paths — same pages, same value model, different data plumbing. Pick the one that matches your setup:

  • SharePoint (simplest) — the fastest start on Power BI Pro alone; no Fabric or Premium required.

    Download templateSetup guide

  • Fabric (recommended) — scheduled Spark ingestion into a Lakehouse for the best performance, plus the optional billing and feedback pages; the same build runs on Databricks, Synapse, or Azure SQL.

    Download templateSetup guide

  • Fabric Extended (advanced) — everything in the Fabric path plus the deeper Copilot Studio agent, topic, and CSAT pages; stand up Fabric first, then add this.

    Download templateSetup guide
